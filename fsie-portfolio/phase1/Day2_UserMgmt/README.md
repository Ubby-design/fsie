# Lab: Day 2 — Linux Users, Groups, Permissions & Sudo

## Lab Metadata
- Phase: 1 – Core Systems
- Disciplines: Linux Systems, Security, DevOps Foundation
- Dependencies: Day 1 — Linux Core Setup
- Lab Version: 1.0

## Lab Overview
Create users/groups, configure permissions, and assign sudo privileges to enforce security and enable automation.

## Step-by-Step Execution
1. Add Users — separate operational and admin accounts
2. Add Groups — centralize permission management
3. Configure File/Directory Permissions — enforce least privilege
4. Configure Sudo — controlled admin access
5. Test Permissions — verify correct access
6. Document Users & Permissions — maintain audit trail

## Automation / Configs
- UserMgmt_Setup.sh
- Users_Groups_Permissions.md

## Failure Injection
- Restricted access issues
- Sudo misconfiguration
- Group membership missing
